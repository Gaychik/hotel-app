# Правила разработки проекта Hotel App

## Общие принципы

1. **Язык разработки**: Все комментарии в коде должны быть на русском языке
2. **Технологический стек**: 
   - Next.js 14.2.3 (App Router)
   - React 18.3.1
   - TypeScript
   - Tailwind CSS 3.4.3
   - react-transition-group для анимаций
   - @heroicons/react для иконок
   - date-fns для работы с датами
   - react-day-picker для выбора дат

## Стиль кодирования

### Именование
- Компоненты: PascalCase (например, `Header.tsx`, `SearchButton.tsx`)
- Функции и переменные: camelCase
- Константы: UPPER_SNAKE_CASE
- Типы и интерфейсы: PascalCase с суффиксом `Type` или `Props`

### Форматирование
- Отступы: 2 пробела
- Максимальная длина строки: 100 символов
- Всегда использовать точку с запятой
- Фигурные скобки открываются на той же строке

### Комментарии
- Все комментарии пишутся на русском языке
- Комментарии к функциям должны описывать что делает функция, а не как
- Сложные участки кода должны сопровождаться поясняющими комментариями

## Работа с компонентами

### Создание новых компонентов
1. Размещать в соответствующей директории (`components/layout/` или `components/ui/`)
2. Использовать TypeScript для типизации props
3. Экспортировать как default export
4. Добавлять комментарий с описанием компонента

### Стилизация
- Использовать Tailwind CSS классы
- Избегать inline стилей
- Для сложных стилей использовать CSS модули если необходимо
- Соблюдать дизайн-систему проекта

## Шрифты

Проект использует следующие шрифты через Google Fonts:
- **Inter** - основной шрифт (sans-serif)
- **Karantina** - для логотипа и заголовков
- **Istok Web** - для подзаголовков
- **Source Serif 4** - для акцентного текста

Переменные CSS:
- `--font-inter`
- `--font-karantina` 
- `--font-istok-web`
- `--font-source-serif-pro`

## Работа с состоянием

- Использовать useState/useEffect только в клиентских компонентах (`"use client"`)
- Для глобального состояния рассмотреть Context API или Zustand
- Состояние форм хранить локально в компонентах

## Маршрутизация

- Использовать Next.js App Router
- Динамические маршруты создавать через папки `[param]`
- Страницы размещать в директории `app/`

## Тестирование

- Писать unit тесты для чистых функций
- Использовать Jest и React Testing Library
- Тестировать ключевую бизнес-логику

## Деплой

- Приложение готово к деплою на Vercel
- Следовать официальной документации Next.js по деплою

## Git workflow

- Создавать feature ветки от main
- Коммиты на русском языке
- Pull Request с описанием изменений
- Code review обязательна

## Производительность

- Оптимизировать изображения
- Использовать lazy loading где возможно
- Минимизировать количество перерендеров
- Кэшировать данные при необходимости
